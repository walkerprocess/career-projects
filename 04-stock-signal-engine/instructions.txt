🤖 AI 주식 추천 시스템 - 파일별 역할 및 아키텍처 가이드
================================================================

📁 파일별 역할 설명
==================

🔧 핵심 애플리케이션 파일
------------------------

1. stock_main.py (64KB) - 메인 웹 애플리케이션
   역할: Streamlit 기반 웹 인터페이스
   기능: 
   - 사용자 입력 (투자금액, 섹터 선택, 리스크 설정)
   - 실시간 주식 추천 결과 표시
   - 포트폴리오 상태 모니터링
   - 뉴스 기반 매수 이유 분석
   - 거래 전략 계산 및 시각화

2. stock_analyzer.py (53KB) - 주식 분석 엔진
   역할: 핵심 분석 로직 담당
   기능:
   - Yahoo Finance 데이터 수집
   - 기술적 지표 계산 (RSI, 이동평균 등)
   - 뉴스 수집 및 분석 (30개 뉴스 소스)
   - 주식 점수 계산 (100점 만점)
   - 뉴스 우선순위 주식 선별

3. portfolio_manager.py (27KB) - 포트폴리오 관리
   역할: 포트폴리오 상태 관리 및 리밸런싱
   기능:
   - 현재 포트폴리오 로드/저장
   - 성과 계산 및 추적
   - 자동 리밸런싱 제안
   - 추천 히스토리 관리

🛠️ 지원 및 유틸리티 파일

4. daily_runner.py (12KB) - 자동 실행 스크립트

   기능:

7. stock_trader.py (15KB) - 거래 로직

🔄 데이터 플로우
===============

1. 📱 사용자 입력 (투자금액, 섹터, 리스크)
   ↓
2. 🔍 stock_analyzer.py가 240개+ 주식 후보 분석
   ↓
3. 📰 30개 뉴스 소스에서 우선순위 주식 선별
   ↓
4. 🎯 뉴스+구매가능+유망성 기반 베스트 2개 선별
   ↓
5. 🤖 GPT-4o로 상세 분석 및 매수 이유 생성
   ↓
6. 💰 거래 전략 계산 (매수수량, 손절가, 목표가)
   ↓
7. 📊 결과를 Streamlit UI로 표시
   ↓
8. 💾 포트폴리오 업데이트 및 엑셀 리포트 생성

🎯 핵심 특징
===========

• 뉴스 기반 우선순위: 30개 뉴스 소스 분석으로 최적 주식 선별
• 베스트 2개 추천: 체크박스 없이 깔끔한 인터페이스
• 실시간 분석: Yahoo Finance + GPT-4o 조합
• 자동화: 매일 자동 실행 및 포트폴리오 관리
• 캐싱: API 호출 최적화로 속도 향상
• TLS 우회: curl_cffi로 Rate limit 문제 해결

🚀 실행 방법
===========

1. 웹 인터페이스 실행:
   py -m streamlit run stock_main.py

2. 자동 실행 설정:
   - Windows: run_daily.bat을 작업 스케줄러에 등록
   - Linux/Mac: crontab에 daily_runner.py 등록

3. 시스템 테스트:
   python test_system.py

📋 주요 기능
===========

• 실시간 뉴스 분석 (30개 소스)
• GPT-4o AI 분석
• 기술적 지표 (RSI, 이동평균)
• 투자 금액 맞춤 필터링
• 섹터별 분석 (10개 섹터)
• 자동 포트폴리오 리밸런싱
• 엑셀 성과 리포트
• 캐시 기반 성능 최적화

⚠️ 주의사항
==========

• 이 추천은 AI 분석 결과이며, 투자 결정은 본인의 판단에 따라야 합니다
• 과거 성과가 미래 수익을 보장하지 않습니다
• 투자 전 충분한 리서치를 진행하시기 바랍니다
• 분산 투자를 통해 위험을 관리하세요
• 자동 리밸런싱은 제안사항이며, 실제 거래는 수동으로 진행해야 합니다

📞 문제 해결
===========

• OpenAI API 오류: .env 파일에서 API 키 확인
• yfinance Rate limit: yfinance_patch.py가 자동으로 처리
• 캐시 문제: cache/ 폴더 삭제 후 재시도
• 포트폴리오 오류: portfolio_data/ 폴더 확인
